Monoalphabetic单表密码

基于替代原则（substitution）。以下密码都是基于单表替代原则。

凯撒加密（Caesar）

略

关键字加密（Keyword）

  • 加密过程
    • 关键字去重（“success”-> “suces”）
    • 先列出关键字，然后按照字母表顺序将剩下的字母按顺序排列，组成新的字母表
    • 按照对应关系一一加密
      • 破解方法
      • 频率分析：字母频率分析（ETAONIRSHDLUCMPFYWGBVJKQXZ）字母组合频率分析

        仿射密码（Affine）

  • 加密过程
    • 为每一个字母分配一个数字(0-25)
    • 在0到25之间选择两个数字a和b，要求a和26是互素的（欧几里得除法）
    • c = ap +b其中p是明文字母，c是密文字母。
  • 解密
    • p=a的逆元*(c-b) mod 26
  • 破解
    • 字母频率分析
    • 列出二元一次方程并求解

Polyalphabetic Cipher 多表密码

基于替代原则，但是每一个字母可以被加密为不同的密文，每一个密文可以代表不同的明文字母。以下密码都是基于多表替代原则。

维吉尼亚密码Vigenere cipher

加密如图

playfair cipher

  • 加密
    • 选取关键字
    • 先写关键字，字母表中剩下的字母（除了j）按顺序排列。所有字母组成5x5的矩阵。
    • 按照以下规则进行加密前的预处理：
      • j被替换成i
      • 同样字母之间插入特定字符
      • 奇数个数字符的字符串添加一个特定字符
    • 按照以下加密规则加密：
      • 两个字母同行，两个都分别替换成他们右边的字母
      • 两个字母同列，两个都分别替换成他们下面的字母
      • 不同行不同列，替换成以那两个字母的连线为对角线的矩阵的另外一条对角线两端的字母
  • 破解：
    • probable word attack(可能词攻击)基于已知明文攻击（known-plaintext attack）
    • 检验规则：
      • 明文和密文的对应字符不可能相同
      • er和re都会被相同但排列不同的字符对替代
      • 是否能组成矩阵
    • 每次移动两个字符

  • 特征：

    • 每一个字符只能被5个其他字符替代（同行的4个和下面的一个）
    • 密文中不可能出现连续两个相同字母

      double playfair cipher

  • 加密：

    • 两个关键字，两个矩阵
    • 规定明文周期，然后形成n行周期列的矩阵，如果不足就先等分，中间用低频字符来填充（?），如果是奇数字符就添加一个低频字符
    • 同行的，字符都在各自的矩阵中往右边移动一位，然后交换到对方矩阵中，进行下一次加密，再交换到对方矩阵中，密文为第一个矩阵中的对应字符和第二个矩阵中的对应字符
    • 同列的向下移动一位，其他和同行的规则一样
    • 不同行不同列的，把两个矩阵拼接在一起之后矩阵交换，然后字符交换，然后再次矩阵交换，再次字符交换，得到最终密文
  • 和playfair的区别
    • sr和rs不一定会有同样的相反密文串
    • 连续字母可以不添加低频字母就可以加密
    • 连续两个相同字母可以出现在密文中

总结：以上密码都是基于替代原则。接下来的密码是基于置换原则的。
